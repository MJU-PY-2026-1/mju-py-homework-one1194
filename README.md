# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 축구 감독이 되어 선수들을 관리하고 정규 시즌을 치러 경기를 수행하는 시스템
<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 사용자가 축구팀의 감독이 되어 선수단을 관리하고 경기를 치르는 '구단 운영 시뮬레이션'입니다. 사용자는 자본금(예: 100억 원)을 가지고 게임을 시작하며, 상점에서 실존 선수를 모티브로 한 선수(예: 손흥민 - 공격 90, 속도 93, 수비 50)를 영입하거나 방출할 수 있습니다. 추가로 직접 타이핑하여 선수를 생성할 수도 있어 내가 직접 선수가 되어 뛰어볼 수도 있습니다. (예: 제갈원 - 공격 82, 드리블 78) 선수 데이터는 파이썬의 리스트를 활용해 '나의 스쿼드'로 관리됩니다. 정규 시즌이 시작되기 전 이적 시장을 통해 선수들을 영입하거나 방출할 수도 있습니다. 
스쿼드가 완성되면 '경기 치르기' 메뉴를 통해 상대 팀과 대결합니다. 시스템은 출전 선수들의 총합 공격력과 수비력을 계산하고, 랜덤 변수를 더해 경기 결과를 출력합니다. 승리 시 상금과 선수 경험치를 얻어 능력치가 상승(S, A, B, C 등급 부여)하지만, 체력이 깎이는 조건 로직이 포함됩니다. 최종적으로 메뉴를 통해 선수 영입, 스쿼드 관리, 경기 진행을 자유롭게 오가며 최고의 구단을 만드는 것이 목표입니다.

<br>

## 3. 예상 기능 및 메뉴 (최소 5개)
1. 선수 능력치 확인 시스템 
2. 내 구단 스쿼드 관리 및 재정 조회
3. 선수 영입 및 방출 (이적시장)
4. 선수단 훈련 (스탯 상승, 체력 감소)
5. 경기 진행 (경기 결과 및 보상 시뮬레이션)
6. 선수 부상/체력 회복소 (휴식 기능)
7. 게임 종료

    * **적용 내용:** 이름과 포지션은 문자열(str), 공격력/수비력/체력은 정수형(int), 이적료나 평균 평점은 실수형(float)으로 명확히 분리하고, 이를 리스트(List)로 묶어 전체 스쿼드를 관리함으로써 데이터 접근성과 확장성을 높임. input() 함수를 통해 감독(사용자)으로부터 직접 이름과 능력치를 입력받아 새로운 선수 딕셔너리를 생성하고 스쿼드 리스트에 추가(append)하는 기능을 구현함. if-elif-else을 통해 승/무/패 결과를 도출하는 현실적인 경기 시뮬레이션 로직을 구현함.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
  * 이번 과제는 축구 구단의 이적 시장 시스템 구현으로, 사용자로부터 신인 선수들의 데이터를 입력받아 리스트에 저장하고, 가중치 연산을 통해 등급을 판정하는 프로그램을 구축하는 것입니다. 특히 for 반복문과 if-elif 중첩 조건문, or 논리 연산자, 리스트 조작 등을 종합적으로 활용하여 상황에 맞는 영입 심사와 결과를 도출하는 지능형 로직을 완성해야 합니다.
  * input() 함수를 통해 감독 이름, 구단 예산, 선수의 포지션(FW/MF/DF/GK) 및 6대 스탯(슈팅, 패스, 스피드, 피지컬, 수비, 민첩성)을 입력받아 다차원 리스트(market_players)에 저장.
  * 포지션별 맞춤형 가중치 공식을 적용하여 선수의 종합 능력치(OVR) 산출:
(ex : 공격수(FW) = (슈팅 * 0.35) + (스피드 * 0.2) + (민첩성 * 0.15) ... )
  * if-elif-else 조건문을 사용하여 산출된 OVR 값에 따라 S등급(월드클래스)부터 C등급(보류 권장)까지 4단계의 세밀한 등급 판정 로직 추가.
  * 중첩 if문을 사용하여 구단의 잔여 예산을 확인(예산 >= 이적료)하는 영입 심사를 거치며, or 논리 연산자를 활용하여 이적료가 1500억 이상이된 경우 '대형 계약 성사! 구단주가 기뻐합니다'라는 특별 칭호/이벤트 메시지 출력 기능 구현.
    
### **🤖 AI 파트너십 과정**

1. **내용 1: 길드 매니저 컨셉 및 로직 도출**
    * **프롬프트 요약:** "내가 축구 감독이 되어서 선수들을 직접 관리하는 프로그램을 만들거야 대학교 1~2학년 수준에서 리스트와 조건문을 활용해 등급을 매기는 재미있는 로직을 짜줘"
    * **적용 내용:** AI가 제안한 '가중치 OVR 산출'
    * 공격수(FW): (슈팅 * 0.35) + (스피드 * 0.2) + (민첩성 * 0.15) + (패스 * 0.15) + (피지컬 * 0.1) + (수비 * 0.05)
      미드필더(MF): (패스 * 0.35) + (민첩성 * 0.2) + (스피드 * 0.15) + (슈팅 * 0.1) + (수비 * 0.1) + (피지컬 * 0.1)
      수비수(DF): (수비 * 0.4) + (스피드 * 0.2) + (피지컬 * 0.2) + (패스 * 0.1) + (슈팅 * 0.05) + (민첩성 * 0.05)
      골키퍼(GK): (수비 * 0.4) + (민첩성 * 0.25) + (패스 * 0.15) + (피지컬 * 0.15) + (스피드 * 0.05) + (슈팅 * 0.0)
      
    * 방식과 선수들마다 특징에 따라 명칭을 붙여주는 복합 조건 아이디어를 실제 코드에 반영함.
    * stats = [슈팅, 패스, 스피드, 피지컬, 수비, 민첩성]
      if stats[2] >= 90 and stats[3] <= 60: 치달 장인 (유리몸)
      elif stats[1] >= 85 and stats[5] >= 85: 중원의 마술사
      elif stats[0] >= 85 and stats[1] <= 50: elif stats[0] >= 85 and stats[1] <= 50:
      elif stats[3] >= 90 and stats[4] >= 90 and stats[2] <= 50: 통곡의 벽(느림보)
      elif min(stats) >= 80: 육각형 굴리트 
      
2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수/딕셔너리)에 맞춰서, '축구 감독' 컨셉의 프로그램이 어떻게 발전할 수 있을지 4단계 기능 확장 로드맵을 제안해줘."
    * **적용 내용:** 단순 입출력을 넘어, **'스쿼드 등록 시스템 (1차) → 선수 정밀 진단 시스템 (2차) → 구단 사무소 오픈 및 시즌 진행 (3차) → 본격 구단 매니징 시스템 (4차)'**로 이어지는 논리적인 확장 로드맵을 확립함.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 포지션과 무관하게 동일한 잣대로 선수를 평가하는 문제(논리적 오류)**
    * **원인:** 초기에는 선수의 능력치(OVR)를 구할 때 단순히 6개의 스탯을 모두 더한 뒤 6으로 나누는 단순 평균(sum() / len()) 방식을 사용함. 이로 인해 공격수인데 수비만 잘해도 높은 점수를 받는 등 현실성이 떨어짐..
    * **해결:** 선수 등록 시 '포지션(FW, MF, DF, GK)'을 추가로 입력받도록 수정함. 이후 if-elif 조건문을 사용하여 포지션에 따라 핵심 스탯에 더 높은 비중을 두는 '가중치 연산 공식(예: 공격수는 슈팅 0.35, 민첩성 0.15 등)'을 각각 다르게 적용하여 훨씬 정교하고 현실적인 스카우트 시스템을 완성함.
2. **문제 2: 최종 명단에 선수 이름이 출력되지 않는 현상**
    * **원인:** 이적 시장 종료 후 최종 스쿼드를 보여줄 때, 리스트의 길이를 구하는 len(my_team) 함수만 사용하여 단순히 영입한 선수의 '숫자(명수)'만 출력되도록 작성됨.
    * **해결:** 영입한 선수들의 이름을 한눈에 보여주기 위해 최종 출력 단에 for 반복문을 추가함. my_team 리스트 안에 저장된 선수 이름 데이터(문자열)를 하나씩 순차적으로 꺼내어 깔끔하게 화면에 출력되도록 개선함.
